## Why Split?

The original CHANGELOG.md grew to over 284KB and 7,453 lines, making it difficult to navigate and exceeding file size limits in some tools. Splitting by major version:

✅ Keeps main changelog readable and under 150KB
✅ Preserves all historical information
✅ Organizes by development phase
✅ Makes it easier to find specific version information
